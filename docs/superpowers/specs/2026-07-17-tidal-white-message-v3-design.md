# 消息与房间「潮汐白页 2.0」V3 设计说明

## 目标

将消息首页、个人聊天、隐藏工具面板、私密房创建、通知列表和房间管理统一成一套清透、克制、可信赖的微信小程序界面。业务逻辑沿用现有关系与房间规则，本轮只修正信息结构、视觉系统、操作反馈和原型交互完整度。

## 用户场景

用户多在夜间或碎片时间进入消息页。他们需要快速分辨通知、私聊与房间，继续刚才的关系，而不是先理解复杂功能。页面应像稳定的白色任务空间，保留轻微海岸气质，但不把海景、光效或大面积蓝色带入消息任务。

## 方案选择

采用「潮汐白页 2.0」：苹果式克制结构与项目海蓝品牌色结合。

- 页面以接近白色的冷中性底为主，海蓝只用于当前操作和关系提示。
- 会话与通知使用连续列表、细分隔和留白，不再把每一行做成悬浮卡片。
- 圆角只用于输入框、图标按钮、弹窗和必要状态标签，普通列表不使用大圆角。
- 动效只解释展开、切换、创建、邀请和解散等状态变化。

## 信息结构

### 消息首页

1. 页面标题。
2. 三个通知入口组成安静的连续快捷区：互动通知、邀请与申请、系统通知。
3. 最近会话使用连续列表，个人会话和房间通过头像形态及小型标签区分。
4. 左滑操作逻辑保持不变，展开底色延续已确认的浅白方案。

### 个人聊天

1. 顶部只显示返回、用户名称和更多操作。
2. 消息正文增加真实时间提示、头像层级和更舒适的消息气泡比例。
3. 底部固定顺序：麦克风、输入框、发送、加号。
4. 加号面板默认隐藏；展开时输入栏整体上移，面板从下方进入。
5. 面板分为「媒体」和「互动」两组，通过间距与小标题形成层级，不使用八个同质方格。

### 房间创建

1. 保持用户已确认的居中弹窗，不改为底部抽屉。
2. 弹窗内容顺序为：标题与关闭、类型选择、房间名称、可邀请对象、主操作。
3. 移除卡片套卡片；好友候选使用连续选择行。
4. 创建成功后立即写入最近会话，再进入房间；邀请是创建后的独立动作。

### 通知与房间管理

1. 通知二级页使用连续列表，整行可点击。
2. 邀请和好友申请进入居中确认层，可接受或拒绝。
3. 房间管理的邀请、设置、解散均可点击；解散使用二次确认。

## 视觉规范

- 背景：`#F7FAFC`
- 表面：`#FFFFFF`
- 主文字：`#172536`
- 次文字：`#65768A`
- 关系海蓝：`#358BA9`
- 主操作蓝：`#0A78C8`
- 私密紫：`#775B9F`
- 未读珊瑚：`#EF6467`
- 分隔：`#E7EDF2`
- 常规容器圆角：10–14px；居中弹窗：18px；按钮与标签可使用胶囊形。
- 移动端正文不小于 12px 的原型缩放等效值；真实小程序实现按 14px 基准落地。
- 触控目标不小于 44×44px。

## 状态与反馈

- 创建、邀请、接受、拒绝、清空、删除、拉黑、退出和解散均提供即时状态反馈。
- 创建后最近会话立即出现，刷新后的持久化由后端接口和本地缓存共同保证。
- 邀请等待期间按钮进入禁用状态，并在会话预览同步显示等待回应。
- 危险操作只在最终确认按钮使用珊瑚红，不用大面积红色背景。
- 所有动效支持 `prefers-reduced-motion`。

## 验收范围

- 默认聊天工具隐藏。
- 加号展开和收起无跳动，输入栏位置正确。
- 房间创建弹窗居中、无遮挡、可关闭、可切换一对一/多人。
- 创建后立即进入最近会话。
- 三个通知入口、邀请/申请、房间邀请、设置和解散可点击。
- 左滑、清空未读角标、删除、拉黑逻辑不回退。
- 390×844 与 375×812 两种移动尺寸均无横向溢出。

## 非本轮范围

- 不接入真实语音、相册、视频和即时通信接口。
- 不改动关系权限、好友申请限制、房间生命周期及后台配置口径。
- 不新增公开房创建流程；个人聊天内创建的房间仍为私密房。
