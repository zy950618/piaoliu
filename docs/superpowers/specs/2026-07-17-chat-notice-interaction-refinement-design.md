# 聊天设置与通知交互精简设计

日期：2026-07-17

## 目标

本轮只处理三处已经确认的问题：

1. 精简个人聊天中的设置操作，移除多余的分类标题、卡片嵌套、说明文字和彩色图标。
2. 修正漂流瓶回复通知的目标页面，使其进入瓶子详情并定位到对应回复。
3. 为“邀请与申请”列表增加左滑删除，同时保持通知删除与业务拒绝相互独立。

不修改聊天输入区、玩法工具、房间创建、房间聊天和消息首页的其他视觉与逻辑。

## 一、聊天设置

### 页面结构

右上角“···”继续进入聊天设置二级页。页面只保留一个平面列表，不显示“消息”“会话管理”“安全”等分类标题，也不使用分组卡片。

列表顺序固定为：

1. 消息免打扰
2. 清空聊天记录
3. 删除会话
4. 举报用户
5. 拉黑用户 / 解除拉黑

### 视觉规则

- 每项独占一行，最小触控高度 56px。
- 使用页面白色底和细分割线形成层级，不增加外围卡片边框与阴影。
- 去掉所有方形彩色图标和辅助说明文案。
- 消息免打扰右侧使用开关；举报用户右侧使用箭头。
- 清空聊天记录使用常规深色文字。
- 删除会话和拉黑用户使用克制的危险色文字；解除拉黑恢复为常规强调色。
- 页面左右留白与现有聊天页一致，过渡继续使用横向二级页动画。

### 操作规则

- 消息免打扰：点击整行或开关均切换状态，状态写入当前会话模型。
- 清空聊天记录：点击后显示二次确认；确认后仅清空当前用户看到的消息，保留会话，未读数归零。
- 删除会话：点击后显示二次确认；确认后删除本地消息和最近会话入口。
- 举报用户：进入独立举报二级页，不使用一行 Toast 代替流程。
- 拉黑用户：点击后显示二次确认；确认后禁用双方直接消息发送权限，但保留会话入口和解除拉黑入口。
- 解除拉黑：点击后显示确认；确认后恢复当前会话发送能力。

## 二、漂流瓶回复通知

### 跳转目标

“小雨回复了你的漂流瓶”不再进入与小雨的个人聊天。点击后进入该漂流瓶的详情页。

瓶子详情页包含：

- 原漂流瓶内容与投递信息。
- 回复列表。
- 与通知关联的目标回复。

进入页面后自动滚动到目标回复，并用短暂的浅蓝背景与描边提示当前位置。高亮只承担定位反馈，约 1.2 秒后自然淡出；开启减少动态效果时直接显示定位状态，不执行滚动动画。

### 数据关联

互动通知需要携带以下标识：

- `bottleId`：目标漂流瓶。
- `replyId`：目标回复。
- `actorId`：回复用户。

页面按 `bottleId` 打开详情，再按 `replyId` 查找目标回复。当前原型使用固定数据模拟，后续接口保持相同字段语义。

### 异常处理

- 漂流瓶不存在或已删除：显示“这只瓶子已经回到海里了”，并提供返回通知列表。
- 回复被删除或违规隐藏：仍打开瓶子详情，并提示“这条回复已不可见”。
- 找不到目标回复：停留在回复列表顶部，不错误跳转到个人聊天。

## 三、邀请与申请左滑删除

### 列表结构

仅“邀请与申请”列表启用左滑删除。每条通知由两层组成：

- 前层：通知内容，点击进入邀请或好友申请处理流程。
- 后层：单个“删除”按钮，左滑后露出。

删除按钮宽度 76px，使用浅危险色背景与清晰文字，不使用多个并排操作。

### 手势规则

- 水平移动超过 8px 后进入滑动状态，避免普通点击误触。
- 左滑超过 42px 时自动展开；不足则回弹关闭。
- 同一时间只允许一条通知保持展开。
- 点击列表外部或打开另一条通知时，当前展开项自动收起。
- 垂直滚动优先，纵向位移大于横向位移时不拦截页面滚动。

### 删除语义

删除通知只移除消息中心中的这条通知，不代表拒绝邀请或拒绝好友申请，也不修改业务申请状态。

点击通知内容进入处理流程后：

- 接受或拒绝由现有业务按钮完成。
- 业务处理成功后，该通知可更新为已处理或从列表移除。
- 左滑删除无需二次确认，但提供短暂“通知已删除”反馈。

列表删除到空时显示简洁空状态：“暂时没有新的邀请与申请”。

## 四、状态与数据流

### 聊天设置状态

`directConversations` 继续承载：

- `unread`
- `blocked`
- `muted`
- `deleted`
- `messages`

清空、删除、拉黑和免打扰均从同一会话状态读写，聊天页、最近会话和设置页只渲染该状态，避免重复维护角标或按钮文字。

### 通知状态

通知项从字符串数组调整为带稳定标识的对象：

- `id`
- `type`
- `title`
- `summary`
- `action`
- `bottleId` / `replyId`（仅漂流瓶回复）

左滑删除按 `id` 移除通知视图数据；邀请或申请业务状态不随通知删除改变。

## 五、验证标准

### 聊天设置

- 页面不存在“消息”“会话管理”“安全”分类标题。
- 五个操作按固定顺序形成单行列表。
- 清空后消息为空、会话保留、未读角标归零。
- 删除后聊天关闭且最近会话移除。
- 拉黑后输入区不可发送，解除后恢复。

### 漂流瓶通知

- 点击回复通知不会进入个人聊天。
- 打开瓶子详情后定位到对应回复。
- 返回操作回到互动通知列表。

### 邀请与申请

- 左滑可以稳定露出单个删除按钮。
- 普通点击不会触发滑动删除。
- 删除一条通知不触发拒绝逻辑。
- 删除全部通知后显示空状态。
- 纵向滚动不被水平手势错误拦截。

### 通用检查

- 390×844 微信小程序常用手机视口下无溢出。
- 所有触控目标不小于 44px。
- 浏览器控制台无 JavaScript 错误和警告。
- 减少动态效果模式下没有强制滚动动画。
