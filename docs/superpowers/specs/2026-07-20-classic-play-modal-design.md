# 真心话 / 大冒险经典弹窗视觉规范

## 目标

将聊天内「真心话 / 大冒险」玩法弹窗从功能确认框，调整为更接近 iOS / 微信体系的轻量邀请卡。它要显得圆润、经典、清透，但不改变已确认的聊天、房间、语音和加号面板逻辑。

## 范围

- 仅调整玩法弹窗视觉：真心话、大冒险共用。
- 保留现有玩法逻辑：普通 / 私密、随机题、自己输入、发送邀请、取消。
- 不调整骰子结果、房间管理、好友申请、清空删除拉黑等其他弹窗逻辑。

## 视觉方案

- 弹窗使用白色主卡片，背景遮罩保留轻柔虚化。
- 玩法弹窗内部增加专属容器，避免污染通用 action-card。
- 标题左侧增加极轻的玩法符号，标题更像「发起真心话 / 发起大冒险」。
- 普通 / 私密使用浅色胶囊分段控件，选中态只用白底和轻阴影。
- 输入框使用清透浅底，随机按钮保持在输入框右侧，作为图标按钮，不额外占一行。
- 底部按钮仍为左右两列，但取消按钮更轻，主按钮更稳，按钮高度不低于 44px。

## 交互规则

- 点击真心话 / 大冒险后弹出居中卡片。
- 点击随机图标时只替换输入框内容，并保持输入焦点。
- 未输入题目点击发送时，保留原错误提示逻辑。
- 发送后关闭弹窗并追加玩法消息。

## 验收

- 真心话截图不再像表格确认框，整体更圆润、更经典。
- 大冒险继承同一视觉。
- 清空 / 删除 / 拉黑 / 房间管理等 action-card 不出现玩法弹窗专属样式。
- 现有静态契约和浏览器流程不因视觉修改失效。
