# 聊天输入栏与原生键盘对齐设计

日期：2026-07-17  
状态：已实施，并通过静态契约、真实浏览器交互、视觉截图和完整消息流程回归

## 目标

本轮只统一漂流瓶回复、单人聊天和房间聊天的输入习惯，不改变消息权限、房间关系、已有八个单聊工具及其业务逻辑。

## 1. 漂流瓶回复键盘

### HTML 原型

- 删除“慢慢说”“我在听”“抱抱你”等自定义候选词，不在输入法区域替用户预设表达。
- 删除当前 QWERTY 全键盘视觉，改为无候选词的中文九宫格示意。
- 九宫格使用常见的数字与拼音字母组合：`1`、`2 ABC`、`3 DEF`、`4 GHI`、`5 JKL`、`6 MNO`、`7 PQRS`、`8 TUV`、`9 WXYZ`；底部保留符号、`0`、删除和完成等必要键位。
- 原型键盘仅用于桌面预览键盘高度、输入条位置和页面压缩过程，不接管真实文本输入，也不响应点击。
- 点击回复、聚焦、内容区压缩、发送后键盘与输入条收起的现有流程保持不变。

### 微信小程序正式实现

- 正式页面调用微信原生输入框和手机系统输入法，不实现自定义软键盘。
- 九宫格或全键盘由用户手机输入法设置决定，产品不得强制切换，也不得伪造系统候选词。
- 输入区继续使用真实键盘高度事件计算位置，兼容安全区域；原型九宫格 DOM 不进入小程序生产实现。

## 2. 单人聊天输入栏

- 麦克风按钮从输入框外部移入输入框胶囊左侧，成为输入框内部的前置操作。
- 麦克风仍保留不小于 44px 的点击区域；视觉图标可以较小，但不能缩小实际触控范围。
- 文本输入内容从麦克风右侧开始，不能与图标重叠；提示文字和已输入内容保持可读。
- 点击麦克风继续调用现有 `toggleVoiceMode()`：文本模式与语音模式切换、拉黑权限判断、工具面板关闭逻辑均不改变。
- 语音模式继续显示“按住说话”，再次点击前置图标返回文本模式。
- 输入栏从左到右为“内嵌麦克风的输入框、发送、加号”；发送和加号的尺寸、颜色及现有事件保持不变。
- 单聊 `+` 面板仍保留八个功能及既定顺序，不重新分组。

## 3. 房间聊天输入栏

- 房间会话可输入状态下，输入栏从左到右统一为“输入框、发送、加号”，加号固定在最右侧。
- 房间加号使用与单人聊天一致的圆形图标、44px 点击区域和打开/关闭状态反馈。
- 点击房间加号后，输入栏按与单聊一致的方式上移，底部展开房间媒体工具区；不再使用居中操作弹窗承载普通媒体选择。
- 房间媒体工具区只保留图片、闪照、视频三个入口，不出现摇骰子、真心话、大冒险、房间或更多玩法。
- 房间未激活或仍在等待成员加入时，维持现有锁定提示，不显示可操作的发送和加号。
- 房间消息发送、媒体选择 Toast、房间生命周期和成员权限逻辑保持不变。

## 视觉与交互约束

- 三类输入栏继续使用现有“潮汐白页”色彩：白色操作层、浅灰输入底、操作蓝发送按钮、深墨图标。
- 不新增悬浮卡片、候选文案、渐变或装饰动画。
- 输入栏状态变化控制在 150–240ms，并继续支持减少动态效果。
- 麦克风、发送、加号均保留明确的焦点名称和不小于 44px 的触控区域。
- 展开工具区时只调整当前会话内部内容高度和底部间距，不允许页面横向偏移或外层滚动。

## 验收标准

1. 漂流瓶原型键盘中不存在“慢慢说”“我在听”“抱抱你”或 QWERTY 行。
2. 原型显示完整九宫格数字与拼音字母组合，键盘仍为不可交互的视觉模拟。
3. 点击漂流瓶回复后输入框聚焦、页面压缩；发送后键盘和输入条收起，新回复可见。
4. 单聊麦克风位于输入框胶囊内部左侧，点击后仍能进入和退出语音模式。
5. 单聊输入框、发送和加号顺序正确，原有八个工具入口及事件完全不变。
6. 房间聊天加号位于发送按钮右侧，图标与单聊保持一致。
7. 房间加号展开底部媒体区，只有图片、闪照、视频，关闭后输入栏和消息区恢复。
8. 锁定房间不暴露发送与媒体操作。
9. 单聊、房间、漂流瓶回复均无横向偏移、输入遮挡、异常外层滚动或控制台错误。
10. 微信小程序实现说明明确：使用真实系统键盘，不能强制九宫格。

## 本轮不处理

- 不制作微信小程序自定义输入法。
- 不增加房间语音、玩法或新的媒体类型。
- 不修改单聊八个工具的业务、顺序、图标和权限。
- 不调整会话列表、通知、房间邀请、拉黑、清空或删除逻辑。
