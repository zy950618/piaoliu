# 设计规范

## 视觉方向

关键词：高级、克制、轻社交、匿名安全感、女性友好。参考 Apple 的自适应颜色和材质层次，以及 Xiaomi 产品页的大留白和少量橙色强调。整体使用移动端居中容器，避免 H5 宽屏像后台页面。

## 图标规则

首版使用跨端稳定的单字识别图标：

- 瓶：漂流瓶
- 树：树洞
- 广：广场
- 近：附近的人
- 金：金币钱包
- 证：真人认证
- 礼：礼物
- 黑：黑名单
- 会：会员
- 信：消息

后续进入视觉设计阶段再替换为正式 iconfont 或 SVG 图标。

## 页面布局

- 漂流瓶页：使用柔和月光海面，不使用大面积深黑或纯白背景；动画铺满首屏，主界面只保留标题、筛选和“捞/扔”两个动作。捞瓶和扔瓶都通过弹窗承载内容，不在主界面堆叠卡片。
- 漂流瓶页的“捞/扔”必须保留轻量角标次数，例如“捞”按钮右上角显示剩余次数，不在主界面铺开五类次数。
- 树洞页：薄荷/自然感，突出安全、匿名和共鸣。
- 广场页：暖色和玫瑰色，突出公开动态和话题。
- 附近的人页：薄荷色，突出粗略距离、认证和安全互动。
- 游戏页：中性黑白卡片，突出常规/私密模式切换。
- 钱包页：双卡片区分充值金币和魅力值。
- 认证页：认证状态、拉新赠会员、收益规则必须第一屏可见。
- 底部 Tab 为：瓶子、广场、游戏、树洞、我的。
- 游客端不出现后台入口。
- 签到只在我的中心展示。
- 不在用户端展示“认证用户列表”或“营销吸收点”说明文案；这些能力体现在功能入口、标签、活动和转化路径中。

## 筛选控件

- 筛选必须带小图标，不允许只有突兀文字胶囊。
- 城市、性别、年龄段分别使用“城 / 性 / 龄”识别图标。
- 筛选默认收起，只在用户主动点击后以弹窗或轻量横条展示。
- 瓶子页主界面只显示一个“筛选”入口，完整筛选项放入弹窗。
- 瓶子页筛选弹窗必须居中展示，包含“取消”和“保存”。保存只应用到当前页面状态，不写入持久偏好。

## 瓶子造型

- 瓶子必须有瓶塞、瓶口、瓶颈、玻璃高光、瓶身、纸条/标签，不能只用简单椭圆和矩形拼接。
- 瓶身使用半透明浅青绿玻璃质感，避免浑浊深色。
- 漂浮动画要轻，捞瓶动画要明显：光束、扩散圆环、瓶子上浮共同表达“捞到”。
- 捞到瓶子后的弹窗必须展示作者头像、昵称、VIP 标识、性别、年龄段、城市、认证标签，再展示瓶子正文和回应动作。
- 扔瓶弹窗必须支持本次选择“谁可以捞”和地区；这些选择只属于本次发布，关闭或发布后重置，不记录成下一次默认值。

## 预览与版本收集

- 每次需要预览时，先记录当前版本状态，再截图预览。
- 版本状态至少包含：页面路径、主要改动、截图文件名、验证命令结果。
- 截图命名使用功能和版本号，例如 `runtime-bottle-fullscreen-v4.png`。
- 如果用户指出设计问题，先阅读当前代码和设计文档，再基于最新版本继续改，不从旧方案回退。
