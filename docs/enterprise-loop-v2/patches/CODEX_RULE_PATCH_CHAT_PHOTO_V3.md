## 1. 替换旧规则：陌生人私聊

### 1.1 旧规则废弃

以下旧表述废弃，不再作为最终产品规则：

- `陌生人不能直接私聊。`
- `好友申请通过后才开放聊天。`
- `私聊只能由好友关系触发。`

### 1.2 新规则：禁止冷启动骚扰，但允许上下文私聊

正确规则：

1. 平台禁止“无上下文冷启动私聊”。
2. 平台允许“明确互动上下文内的陌生人私聊”。
3. 私聊不必须依赖好友关系。
4. 好友关系只是长期关系沉淀，不是私聊的唯一门槛。
5. 私聊是否开放，由“上下文互动 + 双向回应/确认 + 风控状态”共同决定。

### 1.3 可开放私聊的上下文

#### A. 漂流瓶

允许私聊的条件：

1. 用户 A 扔出瓶子。
2. 用户 B 捞到瓶子并提交有效回应。
3. 用户 A 对该回应进行以下任一动作：
   - 回复 B 的回应；
   - 点击“继续聊”；
   - 点击“私聊”；
   - 对该回应做明确接收动作。
4. 系统创建一条带来源的会话：`source_type=bottle_reply`、`source_id=bottle_id`、`reply_id=reply_id`。
5. A 与 B 可以在该上下文会话中聊天，无需先成为好友。

禁止：

- B 捞到瓶子后，在 A 未回应/未确认前无限制私聊 A。
- 任意用户通过用户 ID、头像、主页直接冷启动私聊陌生人。
- 被拉黑、被封禁、风控命中、举报处理中用户继续发起上下文私聊。

#### B. 广场 / 树洞 / 评论

允许私聊的条件：

1. 用户 B 对用户 A 的帖子、树洞、动态或留言进行有效评论/回应。
2. 用户 A 对该评论/回应执行以下任一动作：
   - 回复该评论；
   - 点击“继续聊”；
   - 点击“私聊”；
   - 将该评论标记为愿意交流。
3. 系统创建带来源的会话：`source_type=plaza_comment | treehole_comment`。
4. 会话首屏必须展示来源卡片，避免用户不知道对方从哪里来的。

禁止：

- 评论者在作者未回应/未确认前直接私聊作者。
- 通过广场列表、附近的人列表、用户详情页直接骚扰。
- 通过批量脚本、频繁评论、频繁试探来绕过私聊门槛。

#### C. 游戏 / 房间 / 扩列

允许私聊的条件：

1. 双方进入同一游戏上下文，例如真心话、大冒险、普通房间、私密房间或扩列匹配。
2. 双方完成一次明确互动，或至少一方发起继续聊且另一方确认。
3. 会话带来源：`source_type=game_room | match_expand | private_room`。
4. 私密房间必须有更强的风险提示、举报入口、退出入口和聊天记录审核能力。

### 1.4 好友关系的新定位

好友关系不是私聊唯一条件。

好友关系用于：

- 长期关系沉淀；
- 消息列表优先级；
- 更低频控限制；
- 更完整资料可见性；
- 后续关注流、回访、礼物、活动邀请等权益。

非好友上下文会话限制：

- 默认频率更低；
- 首次会话必须带来源卡片；
- 可设置会话有效期；
- 支持“一键不再接收”；
- 支持拉黑、举报、清空上下文；
- 命中风险词时自动进入审核或限流。

## 2. 替换旧规则：私密照片审核

### 2.1 旧规则废弃

以下旧表述废弃：

- `私密照片必须先审后展。`
- `所有私密照片都必须人工审核后展示。`

### 2.2 新规则：智能审核优先，风险分级，人工复核兜底

正确规则：

1. 私密照片采用 AI 智能审核优先。
2. 低风险、模型高置信、符合平台规则的内容可自动通过。
3. 中风险、低置信、边界内容进入人工复核。
4. 高风险内容自动拒绝或冻结，并进入复核队列。
5. 用户举报、风控异常、收益异常、重复违规，会触发二次审核。
6. 人工审核只处理模型无法可靠判断、高风险、举报、抽检和申诉，不做全量人工先审。
7. 审核必须保留模型证据、动作原因和审计日志。

### 2.3 风险分级

#### low_risk：自动通过

满足：

- 非露骨；
- 无敏感隐私；
- 无证件、银行卡、住址、电话等可识别隐私；
- 无未成年人疑似；
- 无诱导充值、外部联系方式、诈骗暗示；
- 无明显违规姿势或裸露重点区域；
- 模型置信度达到配置阈值。

动作：

- `review_status=approved_auto`
- 可展示；
- 可进入收益规则；
- 写入 AI 审核日志；
- 后续仍可被举报或抽检。

#### medium_risk：人工复核或限流展示

命中：

- 模型低置信；
- 边界性感但不明确违规；
- 存在局部遮挡或上下文不清；
- 图片质量过低；
- 疑似外部联系方式；
- 近期账号存在异常收益或举报。

动作：

- `review_status=manual_review`
- 默认不进入大范围曝光；
- 可进入人工复核队列；
- 后台显示模型标签、置信度和命中原因。

#### high_risk：拒绝或冻结

命中：

- 明显露骨或不适合平台；
- 疑似未成年人；
- 非自愿、偷拍、隐私泄露；
- 证件、银行卡、住址、联系方式等敏感隐私；
- 诱导充值、诈骗、外部导流；
- 多次违规或被集中举报。

动作：

- `review_status=rejected_ai` 或 `frozen_pending_review`
- 不展示；
- 不产生收益；
- 进入后台风险队列；
- 记录审计日志和可申诉入口。

### 2.4 后台需要新增的审核能力

后台私密照片审核应包含：

- AI 审核结果；
- 风险等级；
- 模型标签；
- 置信度；
- 自动动作；
- 是否人工复核；
- 用户举报数；
- 收益影响；
- 申诉状态；
- 审核历史；
- 操作人、操作时间、前后状态、原因。

### 2.5 验收标准

私密照片审核不能只看“能上传”。

必须验证：

1. 低风险图片自动通过并记录 AI 审核日志。
2. 中风险图片进入人工复核队列。
3. 高风险图片自动拒绝或冻结。
4. 被举报图片能触发二次审核。
5. 人工审核可覆盖 AI 结果。
6. 所有状态变化有审计日志。
7. 前端能展示明确状态：审核中 / 已通过 / 未通过 / 申诉中。
8. 用户不能通过刷新、重复上传、改文件名绕过审核。
9. 收益金币只允许来自已通过且未冻结内容。
10. 后台能按风险等级、状态、用户、时间筛选。

## 4. 接口契约建议

### 4.1 上下文私聊

建议新增或补充：

- `GET /chat/eligibility`
  - 入参：`target_user_id`、`source_type`、`source_id`、`reply_id?`
  - 返回：是否允许开启私聊、原因、来源摘要、限制策略。
- `POST /chat/conversations`
  - 入参：`target_user_id`、`source_type`、`source_id`、`reply_id?`
  - 动作：创建上下文会话。
- `GET /chat/conversations`
  - 返回当前用户会话列表。
- `GET /chat/conversations/{id}/messages`
- `POST /chat/conversations/{id}/messages`
- `POST /chat/conversations/{id}/block`
- `POST /chat/conversations/{id}/report`

错误码建议：

- `CHAT_CONTEXT_REQUIRED`：缺少合法互动上下文。
- `CHAT_NOT_CONFIRMED`：对方未回复或未确认继续聊。
- `CHAT_BLOCKED`：双方存在拉黑关系。
- `CHAT_RATE_LIMITED`：频率限制。
- `CHAT_SOURCE_NOT_FOUND`：来源内容不存在。
- `CHAT_SOURCE_EXPIRED`：上下文已过期。
- `CHAT_RISK_REVIEW_REQUIRED`：命中风控，需要审核。

### 4.2 私密照片智能审核

建议新增或补充：

- `POST /private-photos`
  - 上传或注册私密照片。
- `GET /private-photos/{id}/review-status`
  - 返回 AI 审核状态、风险等级和用户可见文案。
- `POST /admin/private-photos/{id}/review`
  - 人工复核。
- `GET /admin/private-photos`
  - 支持 `risk_level`、`review_status`、`user_id`、`date_range`、`report_count` 筛选。
- `POST /private-photos/{id}/appeal`
  - 用户申诉。

状态枚举建议：

- `pending_ai_review`
- `approved_auto`
- `manual_review`
- `approved_manual`
- `rejected_ai`
- `rejected_manual`
- `frozen_pending_review`
- `appealing`

## 5. 测试要求

### 5.1 上下文私聊测试

必须覆盖：

1. 无来源上下文，陌生人直接私聊失败。
2. 捞瓶回应后，瓶子作者未确认前不能直接私聊。
3. 瓶子作者回复回应或点击继续聊后，可以创建上下文会话。
4. 广场评论后，作者未确认前不能私聊。
5. 作者回复评论后，可以创建上下文会话。
6. 被拉黑后不能创建或继续会话。
7. 会话首条消息必须携带来源卡片或来源 ID。
8. 会话消息命中违规词会进入审核或被拦截。
9. 非好友会话有频率限制。
10. 好友会话可以走更宽松的频控，但仍受风控约束。

### 5.2 私密照片智能审核测试

必须覆盖：

1. 低风险图片自动通过。
2. 中风险图片进入人工复核。
3. 高风险图片自动拒绝或冻结。
4. 人工复核能覆盖 AI 判断。
5. 举报触发二次审核。
6. 被拒内容不能展示，不能产生收益。
7. 审核日志包含模型标签、置信度、动作、原因和时间。
8. 后台能按风险等级筛选。
9. 重复上传同一图片不能绕过审核。
10. 前端展示明确审核状态和失败原因。

## 6. LOOP 验收新增门禁

新增两个门禁：

### G15 上下文私聊门禁

未通过条件：

- 仍要求所有私聊必须先加好友；
- 允许无上下文冷启动私聊；
- 没有来源追踪；
- 没有拉黑/举报/频控；
- 没有聊天风控；
- 没有上下文会话测试。

通过条件：

- 无上下文私聊被拒；
- 有双向互动的瓶子/帖子/游戏上下文可开启会话；
- 好友不再是私聊唯一入口；
- 所有会话有来源、审计和风控；
- 后端测试、前端交互、后台审计均有证据。

### G16 私密照片智能审核门禁

未通过条件：

- 所有照片都走人工先审；
- 所有照片都无审核直接展示；
- 没有风险分级；
- 没有 AI 证据；
- 没有人工复核兜底；
- 没有举报二次审核；
- 没有收益冻结逻辑。

通过条件：

- AI 审核优先；
- 低风险自动通过；
- 中风险人工复核；
- 高风险拒绝或冻结；
- 所有状态变更有审计；
- 后台可筛选审核队列；
- 前端有明确审核状态；
- 收益只来自合规通过内容。
