# Enterprise LOOP Orchestrator

Use when a task spans multiple files, agents, backend/frontend/admin, or production acceptance.

## Steps

1. Read docs and relevant code.
2. Extract immutable rules.
3. Define allowed and forbidden scope.
4. Split work into agents.
5. Require evidence for every completion claim.
6. Stop on failed tests, missing evidence, business-rule violation, or unapproved high-risk action.

## Output

LOOP START and LOOP RESULT blocks with evidence.
