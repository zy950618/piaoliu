# 用户体验与商业增长路线图 v2

日期：2026-06-29

## 1. 用户端体验原则

漂流瓶用户端不是后台，不应堆复杂筛选和表格。目标是：

```text
第一眼知道能做什么
第一次点击有反馈
失败时知道怎么办
隐私边界清楚
社交连接有节奏
商业转化不突兀
```

## 2. 人类审美验收标准

### 2.1 视觉

- 高级、克制、轻社交、匿名安全感、女性友好。
- 背景不刺眼，不大面积纯黑或刺眼渐变。
- 卡片层次清楚，圆角统一。
- 文案短，不堆平台视角话术。
- 主要按钮突出，次要按钮降低干扰。
- 标签少而准，不把用户卡片做成信息噪音。

### 2.2 交互

- 所有高频点击有即时反馈。
- 提交中按钮 disabled 或 loading。
- 失败不丢输入。
- 弹窗不可误关闭。
- 重要动作二次确认。
- 滑块、筛选、发布、留言、点赞没有 H5 双触发。

### 2.3 文案

不合格：

```text
操作失败
系统异常
提交成功
```

合格：

```text
没有符合条件的瓶子，本次不会扣次数
这条留言只会给主人看到，其他人看不到你的身份
提现申请已提交，预计 1-3 个工作日内完成审核
```

## 3. 页面路线图

### 3.1 漂流瓶

P0：

- 捞/扔入口一直可见。
- 无匹配不扣次数。
- 空内容不提交。
- 回应成功后状态清楚。
- 举报/拉黑不是 toast，必须有弹窗和后续状态。

P1：

- 捞瓶动效：光束、扩散、瓶子上浮。
- 瓶子详情层级：作者 → 内容 → 回复 → 安全动作。
- 随机话术按场景分组。
- 加好友申请有状态：已申请、已通过、已拒绝。

P2：

- 关注流。
- 好友回访。
- 漂流瓶主题活动。

### 3.2 广场

P0：

- 信息流读取后端。
- 图片/语音/视频真实渲染。
- 点赞 toggle。
- 留言弹窗轻量。
- 留言详情页完整。
- 隐藏留言隐私严格。

P1：

- 附近动态、礼物榜、新人推荐榜真实逻辑。
- 发布前内容审核提示。
- 媒体上传状态：上传中、转码中、审核中、失败重试。
- 评论举报/拉黑。

P2：

- 话题。
- 热榜。
- 同城活动。
- 关注用户动态。

### 3.3 树洞

P0：

- 匿名安全提示。
- 发布审核状态。
- 共鸣操作不重复。
- 风险词温和提醒。

P1：

- 情绪分类。
- 温和开导话术。
- 举报和求助入口。

### 3.4 游戏

P0：

- 常规/私密区分。
- 次数扣除正确。
- 抽题反馈。

P1：

- 好看的飞船/星空资产池。
- 抽题动效。
- 私密模式安全提示。

### 3.5 我的

P0：

- 签到。
- 认证。
- 钱包。
- 会员。
- 黑名单。

P1：

- 收益规则解释。
- 认证进度和失败原因。
- 拉新奖励进度。
- 提现状态。

## 4. 扩列功能设计

### 4.1 目标

扩列用于增强连接，但不能绕过好友申请规则。

### 4.2 核心功能

```text
扩列卡片
兴趣标签
城市/性别/年龄筛选
同城/附近粗略距离
关注
申请好友
打招呼模板
举报/拉黑
```

### 4.3 接口草案

```text
GET  /social/expand/users
GET  /social/expand/users/{id}
POST /relations/follow
POST /relations/friend-request
POST /reports
POST /blocks
```

### 4.4 验收

- 无明确互动上下文时不能冷启动私聊。
- 双方完成房间、匹配或扩列上下文内的明确互动/确认后，才出现带来源说明的临时聊天入口。
- 被拉黑后不能重复申请。
- 只展示粗略距离。

## 5. 普通房间设计

### 5.1 目标

多人轻互动，适合真心话、大冒险、同城话题。

### 5.2 功能

```text
创建房间
加入房间
退出房间
房间主题
人数上限
房主管理
成员禁言/踢出
消息审核
举报
```

### 5.3 接口草案

```text
POST /rooms
GET  /rooms
GET  /rooms/{id}
POST /rooms/{id}/join
POST /rooms/{id}/leave
POST /rooms/{id}/messages
POST /rooms/{id}/members/{user_id}/mute
POST /rooms/{id}/members/{user_id}/kick
```

### 5.4 验收

- 被拉黑用户不能进入。
- 房间消息进入风控。
- 房主操作有记录。
- 违规房间可被后台关闭。

## 6. 私密房间设计

### 6.1 目标

更强关系门槛下的私密互动，但不能成为无监管区域。

### 6.2 门槛

可配置：

- 好友关系。
- 实名/人脸认证。
- 会员。
- 邀请码。
- 风控正常。

### 6.3 风控

- 敏感词。
- 举报。
- 截图提醒。
- 分享限制。
- 收益行为审计。

### 6.4 验收

- 未满足门槛不能进入。
- 私密内容仍有举报/审核链路。
- 房主不能越权查看用户私密信息。

## 7. 商业增长漏斗

| 漏斗 | 关键指标 |
| --- | --- |
| 冷启动 | 首次捞瓶成功率、首次发布率 |
| 激活 | D1 留存、首次回应率 |
| 社交连接 | 关注率、好友申请率、好友通过率 |
| 内容互动 | 广场发布率、点赞率、留言率 |
| 认证转化 | 认证开始率、认证通过率、失败重试率 |
| 付费 | 会员点击率、充值转化、礼物发送率 |
| 收益 | 私密照片通过率、礼物收益、提现申请率 |
| 安全 | 举报率、审核积压、封禁率、申诉率 |

## 8. 不合格用户体验示例

- 点击无反馈。
- 失败只显示“错误”。
- 发布失败丢输入。
- 筛选过多占据第一屏。
- 弹窗点击遮罩误关闭。
- 标签过多，真实内容反而不突出。
- 用户不知道金币和魅力值区别。
- 认证失败没有原因。
- 隐私规则文案模糊。
