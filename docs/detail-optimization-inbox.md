# 后续细节优化入口

本文件用于收口不应打断后台骨架优先级的细节。任何 Agent 发现新的体验、规则、接口字段或后台效率问题，先追加到这里，再按优先级处理。

## 使用规则

- 后台接口和后台管理完整骨架为 P0，不被本文件中的前台细节阻塞。
- 新增条目必须写明来源、影响范围、建议优先级和是否需要用户确认。
- 已处理条目不要删除，改为标记完成并链接处理历史。
