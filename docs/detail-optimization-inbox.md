# 后续细节优化入口

本文件用于收口不应打断后台骨架优先级的细节。任何 Agent 发现新的体验、规则、接口字段或后台效率问题，先追加到这里，再按优先级处理。

## 使用规则

- 后台接口和后台管理完整骨架为 P0，不被本文件中的前台细节阻塞。
- 新增条目必须写明来源、影响范围、建议优先级和是否需要用户确认。
- 已处理条目不要删除，改为标记完成并链接处理历史。

## O-001 后台字段级契约

- 来源：后台接口骨架计划。
- 优先级：P1
- 状态：待处理
- 影响范围：后台接口、后台管理端、测试。
- 问题：当前 API Contract 主要是路由级清单，缺少响应字段、分页、筛选、错误码和状态枚举。
- 建议动作：
  - 先为 P0/P1 后台接口补字段级契约。
  - 统一分页参数：`page`、`page_size`。
  - 统一列表响应结构和错误响应结构。

## O-002 后台操作日志细节

- 来源：后台审核和配置类操作。
- 优先级：P1
- 状态：部分完成，Mock 审计链路已接入，真实落库和字段完善待处理
- 影响范围：用户状态、内容审核、奖励配置、提现审核、认证复核。
- 问题：审核类操作需要完整审计；当前已能记录登录、退出、配置保存和审核动作，但仍未进入真实数据库。
- 建议动作：
  - 定义操作日志字段：操作人、模块、对象 ID、动作、原因、前状态、后状态、时间。
  - 后台管理端在详情页展示最近操作记录。

## O-003 漂流瓶视觉和交互细节

- 来源：`bottle-v5` 至 `bottle-v8` 预览迭代。
- 优先级：P2
- 状态：待处理
- 影响范围：瓶子页主屏、筛选弹窗、捞到瓶子弹窗。
- 问题：已有阶段性版本，但后续仍可能继续优化动效节奏、瓶子质感、弹窗信息层级。
- 建议动作：
  - 在后台骨架完成后再继续细化。
  - 每次优化继续记录截图、版本号和验证命令。

## O-004 列表筛选接口化

- 来源：产品规则和 API Contract。
- 优先级：P2
- 状态：待处理
- 影响范围：漂流瓶、广场、游戏、树洞、附近的人。
- 问题：筛选字段首版为前端状态，后续需要进入接口查询参数。
- 建议动作：
  - 后台或服务端列表接口统一支持 `city`、`gender`、`age_range`。
  - 明确默认值、空值语义和非法参数处理。

## O-005 管理端效率功能

- 来源：后台搭建计划 P2。
- 优先级：P2
- 状态：部分完成，内容批量通过/下架和详情面板已接入，高级筛选/导出/趋势图待处理
- 影响范围：后台管理端。
- 问题：当前已有基础批量操作和详情面板，但真实管理还需要导出、趋势图、高级筛选和更多业务队列批量处理。
- 建议动作：
  - 按真实管理频率排序。
  - 优先补高频审核队列的批量处理。

## O-006 管理员真实鉴权与角色权限落地

- 来源：继续完成阶段目标。
- 优先级：P0
- 状态：部分完成，Mock 鉴权、未登录拦截和角色依赖占位已接入，生产级账号和完整权限矩阵待处理
- 影响范围：后台接口、后台管理端、测试、审计日志。
- 问题：当前阶段性骨架仍以 Mock token 为主，真实管理员表、密码哈希、token/session 持久化和菜单权限尚未接入。
- 建议动作：
  - 先定义管理员、角色、权限的数据模型和种子管理员策略。
  - 后端接口统一校验登录态和权限。
  - 前端后台管理端按角色展示菜单和操作按钮。

## O-007 数据库与 Alembic 迁移占位

- 来源：继续完成阶段目标。
- 优先级：P0
- 状态：部分完成，SQLAlchemy/Redis/Alembic 占位和初始迁移已存在，真实连接和迁移执行待处理
- 影响范围：后台接口、审计日志、配置、管理员账号、测试环境。
- 问题：后台真实运行需要把当前占位从 Mock store 迁移到 PostgreSQL、Redis、SQLAlchemy async 和 Alembic。
- 建议动作：
  - 安装并锁定后端依赖，配置 PostgreSQL/Redis 连接。
  - 管理员、角色、权限、审计日志、后台配置优先建表。
  - Redis 只承担会话、短期令牌、频率限制和幂等键，不替代审计持久化。

## O-008 统一错误码与后台错误反馈

- 来源：继续完成阶段目标。
- 优先级：P1
- 状态：部分完成，后端统一错误响应已接入，字段级错误码和前端细分反馈待处理
- 影响范围：后台接口、前端后台管理端、接口契约测试。
- 问题：当前后端已有统一错误结构，但 API Contract 仍需补字段级错误码、状态冲突、重复操作和前端展示细节。
- 建议动作：
  - 定义错误码、HTTP 状态、用户可见文案和日志字段。
  - 后端接口使用统一错误响应。
  - 前端后台页按错误码展示明确反馈。

## O-009 审计日志链路补强

- 来源：继续完成阶段目标。
- 优先级：P1
- 状态：待处理
- 影响范围：管理员操作、审核、提现、认证复核、奖励配置、批量操作。
- 问题：现有骨架已有审计日志展示入口，但真实落库、关联对象、失败操作和批量操作结果仍需补齐。
- 建议动作：
  - 审计记录包含操作人、角色、模块、对象 ID、动作、原因、前状态、后状态、请求 ID、IP 和时间。
  - 详情页展示关联审计记录。
  - 批量操作记录总结果和单项失败原因。

## O-010 后台详情、批量操作与高级筛选入口

- 来源：继续完成阶段目标。
- 优先级：P1
- 状态：部分完成，后台详情面板和内容批量操作已接入，高级筛选和更多业务队列待处理
- 影响范围：用户管理、内容审核、举报处理、提现审核、认证复核、审计日志。
- 问题：当前后台 Dashboard 已有模块摘要、详情面板和内容批量处理，但真实管理需要更多队列详情、权限化批量处理和高级筛选。
- 建议动作：
  - 用户、内容、举报、提现、认证复核、审计日志先补详情页入口。
  - 内容审核、举报处理和提现风控按权限补批量操作。
  - 高级筛选统一支持状态、时间范围、城市、性别、年龄段、风险等级、处理人和金额区间。
