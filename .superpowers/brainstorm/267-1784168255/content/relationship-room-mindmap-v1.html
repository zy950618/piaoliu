关系、消息与房间体系 · 逻辑思维导图

先冻结业务边界，再设计页面。主链始终是“从遇见产生互动，在消息里沉淀关系，通过房间形成更长期的连接”，玩法保持独立，不再挤进房间聊天。

逻辑基线 V1 · 待确认
陌生人关系成长体系 互动事实、私聊授权、好友关系、会话实体和房间成员资格分别保存，任何一个动作都不会误删另一个关系。
关系层

认识，但不强迫升级

状态可以同时存在，不做陌生人到好友的单线等级。

互动事实

漂流瓶回复、广场回复、玩法互动、同房间，仅证明发生过互动。

永久最小凭证不等于好友

私聊授权

记录来源：对方回复、好友通过、其他有效事件。删除会话不会撤销授权。

来源可追踪独立 send gate

好友申请

3天过期；每日10次；同一用户滚动10天最多3次；无拒绝冷却。

默认允许申请双方同意

好友关系

通过后解锁双向私聊，但不制造空会话；第一条真实消息出现后进入列表。

解除好友保留旧会话
消息层

通知聚合，会话分型

同一消息列表承载个人与房间，但操作依据实体类型切换。

顶部三个入口

互动通知、邀请与申请、系统通知。只有待处理事项增加红点。

二级分类结果不常驻角标

普通个人会话

用户头像与昵称。左滑：清空记录、删除会话、拉黑；没有“退出”。

对方新消息可恢复列表

房间会话

房间头像与名称，必须展示公开/私密标签和一对一/多人人数信息。

清空记录退出房间

个人自动清理

默认3天，可选7/15/30天/永不；删除个人历史与入口，不删除关系凭证。

按最后互动计时
房间层

独立聊天空间

房间只支持文字、图片、闪照、视频；真心话等玩法继续独立。

四种组合

公开一对一、公开多人、私密一对一、私密多人；可见范围与人数分两步选择。

默认长期存在

公开加入

创建时选择“直接加入”或“房主确认”。公开房仍允许邀请好友。

申请待房主处理满员停止展示

好友邀请

两类房间都能邀请；好友接受后进入，拒绝则只通知邀请者，不生成会话。

成员邀请由房主开关控制

成员与房主

只有房主能踢人、审批、修改设置、转让和解散。普通成员可退出，无需审批。

房主退出先转让或解散

退出与生命周期

退出者删除自己的记录且永不可重进；他发出的历史消息为其他成员保留。30天无聊天自动解散。

一对一任一方退出即结束
安全层

私密更松，不是无规则

审核强度按空间变化，安全底线和证据链始终存在。

公开房严格审核

敏感色情词发送前阻断；图片、视频采用更严格策略；房名和简介同样审核。

公开可见

私密房宽松审核

成年人正常暧昧表达可通过，但违法、未成年人风险、骚扰和非自愿内容仍阻断。

不可绕过平台底线

拒绝不等于违规

多人拒绝只触发频控；举报触发风险审查。永久封号必须有审核证据。

防恶意举报保留申诉

拉黑是覆盖层

保留好友和授权，只关闭发送；解除拉黑后恢复。与清空、删除、退出完全分离。

不自动退房
数据与验证层

为后续代码留清晰边界

先分实体再写接口，避免继续扩充当前混合会话模型。

核心实体

InteractionReceipt、ChatGrant、FriendRequest、Friendship、Conversation、Room、RoomMember。

ID稳定状态可审计

用户侧状态

ConversationUserState单独保存清空、删除、未读；RoomMember保存邀请、加入、退出、被踢。

避免全局误删

后台动态配置

申请次数、过期时间、房间容量、30天解散、审核阈值、邀请开关均后台可调。

前端只读配置

验证闭环

状态机单测 → API契约 → 页面交互 → 微信开发者工具 → 视觉回归 → 异常与弱网。

每轮只改一个模块防回归

产品主链：每个页面只承担一个关系阶段

已确认业务规则安全边界
遇见 / 广场 / 玩法发现内容和人，产生回复、共同参与、查看资料等真实互动。
→
通知与申请互动通知、好友申请、房间邀请和加入申请集中处理，不制造空会话。
→
普通私聊获得授权后开始对话；聊过不等于好友，好友也不等于必须加入房间。
→
好友关系双向确认后进入长期联系人范围，可被选择邀请进入房间。
→
公开 / 私密房间形成独立长期聊天空间；一对一和多人使用同一房间规则。

我建议直接冻结的默认值

  • 公开房默认允许成员邀请；私密房默认仅房主邀请，房主可开启成员邀请。
  • 公开加入申请与房间邀请默认3天过期；同一个房间同一用户只能有一条待处理记录。
  • 多人房默认上限20人，由后台配置；一对一房固定2人，满员即停止公开展示。
  • 主动退出同一房间不可再次进入；被踢后只有房主解除限制才可能重新邀请。
  • 退出成员的历史消息对其他成员保留，显示原昵称并标注“已退出成员”。

旧模型需要停止继续扩展

  • 聊天工具栏不再出现“公开房 / 私密房 / 临时房”三个直接按钮。
  • 房间不再绑定真心话、大冒险或骰子，玩法独立维护自己的状态。
  • 普通用户会话不再使用“退出”，活跃房间不再使用“删除会话”。
  • 不能用单一 relationship_status 同时表达好友、拉黑、聊天和房间关系。
  • 不能因为拒绝次数直接封号，风险处罚必须有证据链和申诉状态。
